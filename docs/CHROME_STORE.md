```

### Description (Подробное описание)
```
KazEDS заменяет десктопное приложение NCALayer. Сайтам, которые используют электронную цифровую подпись (eGov, kazpatent, банки и др.), больше не нужно, чтобы у пользователя было установлено и запущено NCALayer — KazEDS перехватывает запрос подписи прямо в браузере и проводит подписание на телефоне.

КАК ЭТО РАБОТАЕТ
Расширение эмулирует локальный API NCALayer (WebSocket на 127.0.0.1:13579). Когда сайт запрашивает подпись, KazEDS показывает QR-код. Вы подписываете на телефоне — двумя способами:
• eGov Mobile — официальное приложение: сканируете QR и подписываете штатной ЭЦП;
• KazEDS PWA — загружаете свой ключ .p12, подпись ГОСТ выполняется прямо в браузере телефона.
Результат автоматически возвращается на сайт. Сайту не нужно менять код — он работает со стандартным ncalayer-js-client.

ВОЗМОЖНОСТИ
• Подпись через официальный eGov Mobile (eGov QR)
• Локальная ЭЦП в мобильной PWA (ГОСТ 34.10-2015 через WebAssembly)
• Форматы: CMS/PKCS#7, CAdES-BES, CAdES-T (метка времени НУЦ РК), XMLDSig
• QR-оверлей с переключением eGov Mobile / KazEDS

БЕЗОПАСНОСТЬ
Приватный ключ ЭЦП никогда не передаётся расширению или на сервер — подписание выполняется на устройстве пользователя. Ключ в браузере шифруется AES-256-GCM. Сервер-ретранслятор видит только запрос и готовую подпись, хранит сессию в памяти до 2 минут.

Открытый код: https://github.com/bagdatmussinkz/kazeds
Сайт: https://sign.aitu.uz

## Вкладка «Privacy practices»

### Single purpose (Единственное назначение)
```
Расширение эмулирует локальный API NCALayer (wss://127.0.0.1:13579), чтобы сайты с электронной подписью работали без установки NCALayer, а подписание происходило на телефоне пользователя (через eGov Mobile или мобильную PWA KazEDS).
```

### Permission justification (обоснование разрешений)

**host_permissions «<all_urls>» / content scripts на всех сайтах:**
```
Подпись через NCALayer используется на множестве государственных и коммерческих сайтов (eGov, kazpatent, банки и др.). Расширение должно перехватывать локальный WebSocket NCALayer (127.0.0.1:13579) на любом таком сайте, поэтому content-скрипты внедряются на все страницы. Список сайтов заранее неизвестен и постоянно расширяется.
```

**scripting:**
```
Используется для показа оверлея с QR-кодом на той странице, которая инициировала подпись (инъекция UI оверлея), и для повторной инъекции content-скриптов после установки/обновления расширения.
```

**storage:**
```
Используется только для одного локального флага — включения режима отладочного трейсинга (kazeds_trace), по умолчанию выключенного. Персональные данные и ключи не хранятся.
```

**Remote code:** `No, I am not using remote code` (всё в пакете; cdn.tailwind/шрифты только на сайте-лендинге, не в расширении).
