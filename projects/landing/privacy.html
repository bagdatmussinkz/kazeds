Политика конфиденциальности — KazEDS

Последнее обновление: 8 июня 2026 г.

KazEDS — расширение Chrome, которое заменяет десктопное приложение NCALayer: перехватывает запрос на электронную цифровую подпись (ЭЦП) в браузере и передаёт его на подпись на мобильное устройство (через приложение eGov Mobile или мобильную PWA KazEDS).

1. Единственное назначение

Расширение служит одной цели: эмулировать локальный API NCALayer (wss://127.0.0.1:13579), чтобы сайты, использующие электронную подпись, работали без установки NCALayer, а подписание происходило на телефоне пользователя.

2. Что мы НЕ собираем

  • Приватные ключи ЭЦП. Ключ (.p12) никогда не передаётся расширению или на сервер — подписание выполняется на устройстве пользователя.
  • Пароли, PIN-коды, персональные данные.
  • История просмотров, содержимое страниц, аналитика, рекламные идентификаторы.

3. Какие данные обрабатываются

Для проведения сессии подписания расширение передаёт через защищённый канал на сервер-ретранслятор (Cloud Relay, sign.aitu.uz) только:

  • домен сайта-инициатора подписания;
  • данные, которые сайт запросил подписать (документ/строку), и готовую электронную подпись;
  • служебный идентификатор сессии.

Эти данные хранятся в оперативной памяти ретранслятора временно (до 2 минут на активную сессию) и не записываются в постоянную базу данных.

4. Локальное хранение

Расширение использует chrome.storage.local только для одного флага — включения режима отладочного трейсинга (kazeds_trace), который по умолчанию выключен. Ключи и персональные данные локально не хранятся.

5. Разрешения и зачем они

  • host_permissions <all_urls> + content scripts — подпись через NCALayer используется на множестве государственных и коммерческих сайтов (eGov, kazpatent, банки и др.). Расширение должно уметь перехватить локальный WebSocket NCALayer на любом таком сайте.
  • scripting — для отображения оверлея с QR-кодом на странице, инициировавшей подпись.
  • storage — для флага режима отладки (см. п.4).

6. Третьи стороны

При выборе подписания через eGov Mobile документ передаётся в официальное приложение eGov Mobile Республики Казахстан; его обработка регулируется политикой eGov. KazEDS лишь формирует запрос и передаёт документ.

7. Передача и продажа данных

Мы не продаём и не передаём данные третьим лицам для рекламы или иных не связанных с подписанием целей.
